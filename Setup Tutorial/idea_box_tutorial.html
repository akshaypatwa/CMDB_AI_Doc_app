Section 00

Understanding the Big Picture First

Before touching any tool, understand what we are building and why.

What is the ServiceNow SDK?

SDK stands for "Software Development Kit." Normally, building a ServiceNow app means clicking around in a browser inside ServiceNow's UI, which is slow, hard to version-control, and difficult to automate. The ServiceNow SDK lets you write your app as code on your laptop using modern tools (TypeScript, npm) and then deploy it to ServiceNow automatically.

What is Claude Code?

Claude Code is an AI assistant that lives in your terminal (command line). Instead of writing code yourself, you describe what you want in plain English and Claude Code writes the code for you. It understands your project, reads your files, and makes intelligent decisions.

What is a Skill in Claude Code?

A Skill is a Markdown document that you write once and store in your project. It teaches Claude Code about your specific project — your table names, your rules, your architecture. Without skills, you would have to re-explain your project every time you start a new Claude session.

What Are We Building?

This guide walks you through building the Employee Idea Box — a ServiceNow app where any employee can submit improvement ideas, colleagues can upvote them, and managers can review and approve them. Think of it as a company-wide suggestion box, but structured and trackable.

The app has one table, a simple status workflow, and navigation menus. There is no complex automation — just a clean, deployable app you can have live on ServiceNow in under an hour.

Why Use Claude Code + SDK Together?

Without Claude Code With Claude Code
You write all the TypeScript yourself, configure every field manually, set up all the structures from scratch You describe what you want in plain English. Claude writes the code. You review it. You deploy it.
Requires expert knowledge of ServiceNow APIs and TypeScript The SDK handles the actual push to ServiceNow — no browser clicking required

This guide covers BOTH: the SDK setup AND how to use Claude Code to write all the code using prompts.

Section 01

What You Need to Install

Think of this section as "setting up your workbench." Before you build anything, you need the right tools on your laptop.

Tool 1

Node.js

What it is:Node.js is a runtime that lets your laptop run JavaScript programs. The ServiceNow SDK and Claude Code CLI are both Node.js programs.
Why you need it:Without Node.js, you cannot run npm commands or the SDK.

How to install:

Step 1 — Open your Terminal. On Mac: press Cmd+Space, type "Terminal", press Enter.

Step 2 — Check if Node.js is already installed:

Terminal
node --version

If you see something like v18.17.0, you are good. If you see "command not found," continue below.

Step 3 — Install nvm (Node Version Manager):

Terminal
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

Step 4 — Close and reopen your Terminal window, then install Node 18:

Terminal
nvm install 18
nvm use 18

Step 5 — Verify:

Terminal
node --version
npm --version
Tool 2

The ServiceNow SDK CLI

What it is:A command-line tool made by ServiceNow that handles building and deploying your app.
Why you need it:The SDK compiles your code, packages it, connects to your instance, and uploads everything.

How to install:

Terminal
npm install -g @servicenow/sdk

Verify:

Terminal
now-sdk --version
Tool 3

Claude Code CLI

What it is:The command-line version of Claude (Anthropic's AI). Runs directly in your terminal with access to your files.
Why you need it:This is your AI coding assistant. You describe what you want and Claude Code builds it.

How to install:

Terminal
npm install -g @anthropic-ai/claude-code

Verify and first-time login:

Terminal
claude --version

When you first run claude, it will ask you to log in with your Anthropic account. Follow the prompts — it opens a browser for authentication.

Tool 4

Git

What it is:Version control — tracks changes to your code over time.

Check if installed:

Terminal
git --version

Most Macs come with Git pre-installed. If not: xcode-select --install

Tool 5

VS Code (Recommended Editor)

What it is:A free code editor by Microsoft for reading and reviewing files.

Search "VS Code download" and install from the official Microsoft site.

Section 02

Setting Up Your ServiceNow Instance Connection

Step 2.1: Get Your Instance URL

Your instance URL looks like: https://yourcompany.service-now.com

You should have received this from your ServiceNow administrator. If you are using a personal developer instance (PDI), log in to developer.servicenow.com to find your URL.

Step 2.2: Confirm Admin Access

You need to be an admin on the instance to deploy apps. Log in and confirm you see "Admin" in the top-right corner of the ServiceNow browser UI.

Step 2.3: Add Your Instance to the SDK Keychain

Terminal
now-sdk auth add

The SDK will ask for an alias name (e.g., dev), your instance URL, username, and password. Credentials are stored in your OS keychain — never in project files.

Step 2.4: Verify the Connection

Terminal
now-sdk auth list

You should see your alias and instance URL listed.

Section 04

Understanding Your Project Structure

After now-sdk init, your project looks like this. As you build your app, you will add folders under src/fluent/:

Project Structure 
EmployeeIdeaBox/
│
├── src/
│   └── fluent/                  ← TypeScript files that DEFINE your app structure
│       ├── index.now.ts         ← The master export list — every file must be listed here
│       ├── tables/              ← Your database tables
│       ├── acls/                ← Security rules (who can see and edit what)
│       └── navigation/          ← Menu items in the ServiceNow sidebar
│
├── package.json                 ← Project settings and build commands
├── now.config.json              ← Your app identity (scope, name, scopeId)
└── .eslintrc                    ← Code quality rules
Important — The Barrel File

src/fluent/index.now.ts must export every other Fluent file in your project. Think of it as a table of contents. If you create a new Fluent file but forget to add it here, that piece of your app will be silently skipped during deployment.

Key Concept

Fluent files (.now.ts) are blueprints — they describe the shape of your app: tables, columns, menu items, security rules. They contain no business logic. Think of them as the architect's drawings.

Section 05

Starting Claude Code

Launch Claude Code from Your Project Folder

Terminal
claude

Claude Code starts up and shows you a prompt. It automatically reads your project files to understand what you are working with.

How It Works

When you type a message and press Enter, Claude Code:

  1. Reads your request
  2. Decides what files to create or edit
  3. Shows you what it plans to do and asks for permission
  4. Makes the changes after you approve

Writing Good Prompts

Good prompts are specific (give exact field names and types), purposeful (explain what the thing should DO), and scoped (tell Claude exactly which file to create and what other files to update).

Section 06

Creating Your Skill File

What Is a Skill and Why Create It First?

A Skill file is a briefing document for Claude Code. Every new Claude session starts fresh — without a Skill, you would have to re-explain your project every time. With a Skill file in .claude/skills/, Claude automatically loads your project context at the start of every session.

Real-World Analogy

Imagine hiring a contractor. Without a briefing, they ask basic questions every day. With a proper briefing document, they show up already knowing your requirements, preferences, and constraints.

Prompt to Create Your Skill

Claude Code Prompt
Prompt
Create a skill file at .claude/skills/ideabox.md for this project.

Read now.config.json and package.json to get the real app values, then write a
skill that covers:

1. Project identity: app name, scope prefix, SDK version, auth alias

2. SDK command reference:
   - npm run build: compiles TypeScript to XML, checks for errors, does NOT touch instance
   - npm run deploy: builds + pushes everything to ServiceNow
   - npm run types: fetches type definitions from instance (run when schema changes on instance)
   - npm run transform: pulls current instance state to local files (always commit first)

3. File structure rules:
   - Fluent files (.now.ts) define structure: tables, columns, ACLs, menus
   - Every Fluent file MUST be exported from src/fluent/index.now.ts
   - If you create a file and forget to export it, it will be silently skipped at deploy time

4. App context:
   This is the Employee Idea Box — employees submit improvement ideas, colleagues
   upvote them, and managers review and approve. The app has one table (idea),
   ACLs controlling who can create and edit, and navigation menus.
   All Fluent-only app — no server-side JavaScript needed.
What Claude Code will do

Read your actual package.json and now.config.json to extract real values, then generate a skill file specific to your project.

Section 07

Creating a Reference File

A Reference File is a simple Markdown document with facts Claude can look up — field names, data types, allowed values. Unlike a Skill (which contains rules and instructions), a reference file is a dictionary.

For this app we need just one: a record of all fields in the idea table so Claude always uses the correct names when making changes later.

When to create it

Create the reference file after Prompt A in Section 8 (once the table exists). That way Claude can read the real table definition instead of guessing.

Prompt to Create the Field Reference

Claude Code Prompt
Claude Code
Read src/fluent/tables/idea.now.ts and create a reference document at
docs/field-reference.md.

For each field include: short field name, full scoped field name, data type,
allowed values or max length, and a one-line plain English description.

Also document the status workflow at the bottom:
new → under_review → approved or rejected → implemented

Once created, reference it in any future prompt:

Claude Code
Looking at @docs/field-reference.md, add a new field called target_department
Section 08

Building Your App with Claude Code Prompts

One prompt builds the complete app. Review what Claude creates, approve, then move on to build and deploy.

Claude Code Prompt — Build the Complete App
Prompt
Create the complete Employee Idea Box app in one pass.

Create all four Fluent files below, then add each one's export to
src/fluent/index.now.ts.

───────────────────────────────────────────────────────────────
FILE 1 — Table: src/fluent/tables/idea.now.ts
───────────────────────────────────────────────────────────────
Table purpose: store one record per employee idea submission.
Employees submit ideas, colleagues upvote them, managers review and approve.

Fields:

IDEA CONTENT:
- title: String max 200 chars, mandatory — the idea headline
- description: String max 4000 chars — full explanation of the idea
- category: Choice field with options:
    process_improvement, technology, cost_saving, culture, customer_experience, other
  Default: other

WORKFLOW:
- status: Choice field with options: new, under_review, approved, rejected, implemented
  Default: new
- submitted_by: Reference to sys_user, mandatory — who submitted the idea
- submitted_on: Date — when the idea was submitted
- reviewer: Reference to sys_user — the manager evaluating this idea
- review_notes: String max 2000 chars — feedback from the reviewer
- reviewed_on: Date — when the review was completed

ENGAGEMENT:
- votes: Integer, default 0 — number of upvotes from colleagues
- impact: Choice field with options: low, medium, high
  Description: estimated business impact if implemented

───────────────────────────────────────────────────────────────
FILE 2 — Business Rule: src/fluent/business-rules/set-submitted-on.now.ts
───────────────────────────────────────────────────────────────
Rule: when a new idea record is inserted (on insert only), automatically set
the submitted_on field to today's date if it is not already populated.

───────────────────────────────────────────────────────────────
FILE 3 — ACLs: src/fluent/acls/idea-acls.now.ts
───────────────────────────────────────────────────────────────
ACL 1 — READ: Any authenticated user can read all ideas.
ACL 2 — CREATE: Any authenticated user can submit a new idea.
ACL 3 — WRITE: Users with idea_reviewer or admin role can update ideas
  (change status, add review notes, modify other people's submissions).
ACL 4 — DELETE: Only admin can delete ideas.

───────────────────────────────────────────────────────────────
FILE 4 — Navigation: src/fluent/navigation/app-menu.now.ts
───────────────────────────────────────────────────────────────
Application menu:
- Title: "Employee Idea Box"
- Visible to all authenticated users (no special role required)

Menu items:
1. "All Ideas" at order 100 — shows every record in the idea table
2. "My Ideas" at order 200 — shows only ideas where submitted_by is the
   current logged-in user
3. "Under Review" at order 300 — shows ideas where status equals under_review
4. "Approved Ideas" at order 400 — shows ideas where status is approved or
   implemented
After this prompt

This is a good time to create the field reference document from Section 7 before deploying. Then proceed to Section 9 to build and deploy.

Section 09

Build & Deploy Commands

Step 1 — Build: npm run build

Compiles your TypeScript Fluent files into ServiceNow XML. Checks for errors. Does not touch your instance — nothing is sent anywhere yet.

What success looks like:

Terminal Output 
✔ Compiled 4 modules
✔ No errors found
Build completed in 2.1s

What failure looks like:

Terminal Output (Error) 
✗ Error in src/fluent/tables/idea.now.ts line 22
  Unknown column type 'blob' — did you mean 'string'?

Fix the error in the file, then build again.

Step 2 — Deploy: npm run deploy

The main deployment command. Internally it:

  1. Runs npm run build to compile everything
  2. Packages the output into a ServiceNow Update Set
  3. Authenticates to your instance using the stored keychain credentials
  4. Uploads and commits the Update Set — your app is now live

What success looks like:

Terminal Output 
✔ Table:         x_ideabox_idea
✔ BusinessRule:  Set submitted_on on insert
✔ ACL:           x_ideabox_idea.read (4 rules)
✔ AppMenu:       Employee Idea Box
✔ Deployed successfully in 11.3s

Other Commands (For Reference)

Command What It Does When to Use
npm run types Fetches type definitions from instance When someone changed the schema directly on the instance
npm run transform Pulls instance state to local files When instance was edited via browser — always commit first
Section 10

After Deployment

Step 10.1: Find Your App in ServiceNow

Log into your ServiceNow instance and look in the left sidebar. You should see "Employee Idea Box" with the four menu items you defined. If it does not appear, type Employee Idea Box in the Navigator search bar at the top of the sidebar.

Step 10.2: Create the idea_reviewer Role

Your ACLs reference an idea_reviewer role. Create it on the instance:

  1. Navigate to: User Administration > Roles
  2. Click New
  3. Name: idea_reviewer
  4. Description: Can review and update Employee Idea Box submissions
  5. Save
  6. Assign this role to managers and reviewers via their User records

Step 10.3: Submit a Test Idea

  1. Click All Ideas in the sidebar
  2. Click New
  3. Fill in a title, description, and category
  4. Save — the business rule should automatically populate Submitted On with today's date
  5. Confirm the record appears in My Ideas

Step 10.4: Test the Review Workflow

  1. Open the test idea you just created
  2. Change Status to under_review and assign a Reviewer
  3. Save — the record should now appear under Under Review
  4. Change Status to approved — it should appear under Approved Ideas
Your app is live

That is it. You have a fully deployed, working ServiceNow app built with four Claude Code prompts and the ServiceNow SDK.

Section 11

Tips & Common Mistakes

Tips

Tip 1 — Always Build Before You Deploy

Run npm run build before npm run deploy. It catches errors locally in seconds rather than failing on the instance mid-upload.

Tip 2 — Commit After Each Prompt
Terminal
git add .
git commit -m "Add idea table"

This gives you a restore point. If a later prompt breaks something, you can see exactly what changed with git diff.

Tip 3 — Reference Your Docs in Future Prompts

When asking Claude Code to add or change something later, point it to your field reference:

Claude Code
Looking at @docs/field-reference.md, add a new choice option "urgent"
to the impact field

Common Mistakes

Mistake 1 — Forgot to Export from index.now.ts

What happens: The deploy succeeds without errors but the table or menu does not appear on the instance — silently skipped.

How to fix it:

Claude Code
Check src/fluent/index.now.ts and compare it against all .now.ts files
in src/fluent/ and its subdirectories. Add exports for any missing files.
Mistake 2 — Running transform Without Committing First

What happens: npm run transform overwrites your local files with the instance state. Any uncommitted local changes are lost.

Rule: Always run git status and git commit before running transform. If you do lose changes, check git diff and git checkout . to recover.

Mistake 3 — idea_reviewer Role Not Created

What happens: Users with the right intent cannot update ideas. The ACL references idea_reviewer but if the role does not exist on the instance, the rule silently fails to match anyone.

Fix: Follow Step 10.2 to create the role on the instance after deployment.

Appendix B

Glossary of Terms

Instance
Your company's copy of ServiceNow at a unique URL
Scope
A namespace prefix (e.g. x_ideabox_) that isolates your app from others
Fluent API
ServiceNow's TypeScript way to define app structure locally
Barrel file
index.now.ts — a single file that re-exports everything in the project
ACL
Access Control List — defines who can read, create, write, or delete records
Business Rule
Server-side logic that runs automatically when a record is saved
Update Set
A package of changes deployed as a single unit in ServiceNow
GlideRecord
ServiceNow's API for querying and updating database tables
sys_id
The unique UUID of every record in ServiceNow
npm
Node Package Manager — installs JavaScript libraries
TypeScript
JavaScript with type checking — compiled to plain JavaScript before use
Keychain
Your operating system's secure credential storage (not a project file)
PDI
Personal Developer Instance — a free ServiceNow sandbox for learning
sys_user
The built-in ServiceNow table that stores all user accounts
